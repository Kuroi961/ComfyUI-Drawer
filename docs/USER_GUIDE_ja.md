# ComfyUI-Drawer ユーザーガイド

この文書は ComfyUI-Drawer の機能を README より詳しく説明するためのガイドです。AI に ComfyUI-Drawer の紹介・質問応答をさせるためのコンテキストとしても使えるように、機能の目的、使いどころ、注意点をまとめています。

## ComfyUI-Drawer の考え方

ComfyUI-Drawer は、ComfyUI のノードグラフを別アプリに置き換えるものではありません。ノードグラフは編集可能なまま残し、制作中によく触る操作面だけを下部 Drawer に集約します。

デスクトップでは、Drawer はキャンバス上のワークフローを操作するリモコンのように使えます。モバイルや小さな画面では、Drawer 自体が主な操作画面になります。パラメータ調整、出力確認、モデル選択、検索、辞書、マスク編集、XYZ 検証を同じ場所で扱えるようにすることが目的です。

Drawer 内で動作する各ツールは「ガジェット」と呼ばれます。標準では Home、Deck、XYZ Plot、Gallery、Model Viewer などが入っており、外部カスタムノードから独自ガジェットを追加することもできます。

## まず使う場所

ComfyUI-Drawer をインストールすると、ComfyUI 画面下部に Drawer のタブバーが表示されます。タブを押すと Drawer が開き、もう一度同じタブを押すと閉じます。

主な入口は次の通りです。

- Home: 状態確認、ストレージ概要、更新履歴など
- Deck: ワークフローの主要パラメータ操作
- XYZ Plot: パラメータを変えながら連続生成
- Gallery: output/input/temp のメディア管理
- Model Viewer: models 配下のモデル閲覧と適用
- Settings: テーマ、辞書、検索インデックス、キャッシュなどの設定

## Home

Home は Drawer のダッシュボードです。現在の Drawer バージョン、ComfyUI/Python/PyTorch などのシステム情報、ストレージ使用状況、更新履歴などを確認できます。

ストレージ概要では output、input、models などの容量を確認できます。大量生成や動画生成をしている環境では、Gallery や Model Viewer に入る前の全体把握に便利です。

Home は外部ガジェットからウィジェットを追加できる場所でもあります。サードパーティ拡張が、自分の状態表示やショートカットを Home に置くこともできます。

## Deck

Deck は、ワークフロー内のノードやグループを Drawer 側の操作パネルとして表示するガジェットです。ノードグラフを開いたまま、よく触るパラメータだけを Deck に出せます。

### 表示マーカー

Deck はノードやグループのタイトルに含まれるマーカーを見て表示内容を決めます。

| マーカー | 対象 | 意味 |
|---|---|---|
| `📝` | ノードタイトル | そのノードのウィジェットを Deck のメイン画面に表示 |
| `⚡` | ノードまたはグループタイトル | バイパスを ON/OFF するスイッチを表示 |
| `[ラベル]` | ノードまたはグループタイトル | 同じラベル同士を排他的に切り替える |

ノードは画面上の Y 座標順、グループは X 座標順に並びます。グループ内に表示対象のノードがあれば、グループは Deck 上でもセクションとして表示されます。

### DrawerControls

DrawerControls ノードを使うと、ワークフロー上の複数パラメータをまとめて Deck へ出せます。接続済みの出力だけが表示対象になり、`int`、`float`、`combo`、`bool`、`string` などを扱えます。

文字列出力は次のようにラベルや複数行指定ができます。

```text
string | Label
string | Label | multiline
```

Combo の候補は接続先ウィジェットから取得されます。Deck の表示を整えたいときは、通常ノードに直接マーカーを付けるより DrawerControls にまとめると扱いやすくなります。

## XYZ Plot

XYZ Plot は、A1111 の XYZ Plot に近い考え方で、パラメータを変えながら連続生成するガジェットです。専用の XYZ ノードや追加配線は不要で、現在のワークフロー内にあるウィジェットやバイパス状態をそのままスイープ対象にできます。

### 基本的な流れ

1. X/Y/Z の各軸に対象ノードとウィジェットを指定します。
2. 値のリストや範囲を設定します。
3. プリフライト検証を通します。
4. 生成を開始すると、各組み合わせでキューが実行されます。
5. 軸ラベル付きのグリッド画像が output に保存されます。

テキストウィジェットでは Prompt S/R、つまり Search & Replace 的な指定が使えます。シードはスイープ開始時の状態を基準に固定され、各イテレーションではスナップショットを復元してから軸の値だけを適用します。

### バイパス軸

XYZ Plot は数値や文字列だけでなく、ノードやグループのバイパス状態も軸にできます。

- ノード単体の ON/OFF
- Deck のグループトグル
- グループ排他
- ノード排他

これにより、プロンプト差分だけでなく、サンプラー設定、LoRA、ControlNet、処理枝などの比較にも使えます。

### 実行中の保護

スイープ中は通常のキュー投入を一時的にブロックします。これは外部操作でスイープが混ざらないようにするためです。Drawer 側でも、現在のワークフロー状態を変えてしまう操作を抑制します。具体的には Deck の操作、メディアを LoadImage/LoadImageMask に送る操作、コンテキストメニューからの Mask Editor 起動、Model Viewer のインフォカードからのモデル適用などがロックされます。
サーバー切断が検知された場合は、スイープは中断されます。

## Gallery

Gallery は output、input、temp フォルダ配下のメディアやフォルダを管理するガジェットです。画像、動画、音声、フォルダを扱えます。

### 基本操作

- output/input/temp の切り替え
- フォルダの移動、パンくずナビゲーション
- 名前、日時、サイズでのソート
- ファイル名検索
- ファイルやフォルダのリネーム
- ファイルやフォルダの移動
- ファイル削除
- 新規フォルダ作成
- 画像を LoadImage / LoadImageMask へ送る
- 画像・動画・音声を Lightbox で開く
- 画像からワークフローを開く

削除は可能な環境ではゴミ箱へ送られます。完全削除ではなく、操作ミスへの余地を残す設計です。

### 検索インデックス

Gallery のファイル名検索は検索インデックスなしでも使えます。一方で、prompt、workflow、ノードタイプ、ノードタイトル、カスタムメタデータを検索するには SQLite の検索インデックスを作成します。

検索インデックスは自動で重い処理を始めないよう、ユーザーが明示的に作成します。作成前には簡易計測が行われ、対象ファイル数や推定時間に応じて確認ダイアログが表示されます。

インデックス作成後は、ファイル追加や移動、リネーム、削除を低優先度で同期します。既存ファイルのメタデータは検索用スナップショットとして扱われ、通常の同期では再解釈されません。provider や contributor が追加・変更された場合は、同期時にメタデータ refresh へ切り替わります。

output/input に大量のファイルがある場合、インデックス作成には時間がかかることがあります。Drawer は作成中の進捗を表示し、ユーザーが開始した作成や同期が完了したときはトーストで通知します。作成や同期に失敗した場合は、理由を含むエラーダイアログを表示します。直前の簡易計測結果は可能な範囲で再利用されるため、作成開始直後に同じカウント処理を繰り返しにくくなっています。

### 検索構文

通常検索ではスペース区切りが AND として扱われます。

```text
white hair blue eyes
```

引用符で囲むとフレーズ検索になります。

```text
"white dress"
```

`-word` や `-"quoted phrase"` で NOT 条件を指定できます。

```text
white hair -night
"flower field" -"low quality"
```

特定のノードタイプ内の値だけを検索する場合は `type:...[]` を使います。`[]`は仮想検索欄を表しており、この中身が検索欄のように振る舞います。

```text
type:CLIPTextEncode[white hair -night]
```

特定のノードタイトルで絞り込む場合は `title:...[]` を使います。

```text
title:positive[blue sky]
title:"Prompt A"[school uniform]
```

サードパーティのカスタムメタデータは、contributor が登録されている場合に次の形式で検索できます。

```text
myPlugin[black hair]
myPlugin:tags[black hair]
myPlugin:project[archive A]
```

`namespace[value]` はその namespace 内の全カスタムフィールド検索、`namespace:key[value]` は特定キーだけの検索です。

### 検索対象フィルター

Gallery の検索対象メニューでは、検索対象を次のように切り替えられます。

- ファイル名
- prompt title
- prompt 値
- workflow title
- workflow 値
- カスタムメタデータ

カスタムメタデータの項目は、検索 contributor が登録されている場合だけ表示されます。登録がなければ UI にも出ず、検索対象にも含まれません。

### メタデータ表示

Gallery からメタデータ表示パネルを開くと、ファイル概要、workflow のノード概要、プロンプト値、Raw JSON などを確認できます。

サードパーティがメタデータパネル contributor を登録している場合、そのカスタムメタデータもパネルに表示できます。Drawer 自体はサードパーティの保存形式を固定せず、contributor が「Drawer にどう見せるか」を返す設計です。

## Model Viewer

Model Viewer は ComfyUI の `models` フォルダと、`extra_model_paths.yaml` で追加されたモデルパスをブラウズするガジェットです。

### 対応するモデル種別

checkpoints、loras、vae、embeddings、controlnet、upscale_models など、ComfyUI が扱う主要なモデルフォルダを対象にできます。

### サムネイルとプレビュー

モデルごとにサイドカー画像をプレビューとして表示できます。output 内の画像をモデルプレビューとして設定したり、既存のプレビューを削除したりできます。`.mp4` や `.webm` の動画プレビューにも対応しています。

動画サムネイルや動画/音声メタデータの取得には、利用可能な場合に外部バイナリ `ffmpeg` / `ffprobe` を使用します。`ffmpeg` が利用できない場合、動画サムネイルはプレースホルダー表示になります。

### CivitAI 同期

モデルの SHA256 ハッシュを使って CivitAI の情報を取得できます。`.red` と `.com` の fallback に対応し、モデル情報、プレビュー画像、LoRA の trainedWords などを表示できます。

### ノードマッチング

Model Viewer のインフォカードから、現在のワークフロー内にある対応ローダーノードへモデルを適用できます。通常ノードだけでなく、サブグラフ、Combo Clone、接続済み DrawerControls もスキャン対象です。

## ユーザー辞書、ワイルドカード、コメント

Drawer にはプロンプト入力補完用の辞書サービスがあります。辞書は Settings から管理できます。

### 辞書の種類

- Danbooru タグ辞書: 使用頻度付きタグ CSV
- ユーザー辞書: `tag` から `insert_text` を挿入する CSV
- ワイルドカード: `__名前__` でランダム展開する TXT
- カスタムメタデータキー: サードパーティ dictionary provider が登録した検索補完候補

ユーザー辞書と Danbooru 辞書は、プロンプト入力や Gallery 検索欄の補完に使えます。カスタムメタデータキーは dictionary provider が登録されている場合だけ設定画面に表示されます。

CSV / TXT インポートは 1 ファイルあたり約 5 MB までです。

### ワイルドカード

ワイルドカードは `__名前__` の形式で書きます。有効なワイルドカード辞書に同じ名前があれば、候補から 1 行が選ばれます。ノードや配線は不要です。キュー投入時の prompt payload に含まれる文字列入力が展開対象です。

```text
masterpiece, __style__, 1girl
```

展開はワークフロー内の `seed`、`noise_seed`、`seed_value` を元に決まります。シードが見つかれば同じシードで同じ展開になります。見つからない場合は通常のランダム選択です。

ワイルドカードの再帰展開は行いません。候補内に別の `__name__` が含まれていても、その文字列はそのまま残ります。

### コメント

Drawer は prompt 内のコメントをキュー投入時に取り除けます。ワイルドカードと同様に、ノードや配線は不要で、キュー投入時の prompt payload に含まれる文字列入力が除去対象です。

```text
masterpiece, best quality
// this line is ignored
# this line is also ignored at line start
/* block comment */
```

コメントは実行用プロンプトからは除去されますが、出力画像に埋め込まれる workflow metadata には保持されます。コメント内のワイルドカードは展開されません。

## Mask Editor

Mask Editor は画像のコンテキストメニューから開ける簡易マスク編集 UI です。生成したマスクは `input/drawer_masks` に保存され、`LoadImageMask` ノードへ直接適用できます。

Gallery や Lightbox から画像を開き、必要な部分だけを素早くマスク化したいときに使います。専門的な画像編集ソフトほどの機能はありませんが、ComfyUI 内で完結する簡単な修正に向いています。

## 共通 UI

Drawer のガジェットは、共通 UI サービスを使って一貫した操作感を持ちます。

### Context Menu

右クリックまたはロングタップで開くメニューです。画像を新しいタブで開く、LoadImage へ送る、ワークフローとして開く、ダウンロードする、マスクを作るなどの共通操作を提供します。

ガジェットやサードパーティ拡張は、同じ context menu に独自アクションを追加できます。

### Lightbox

画像、動画、音声を全画面で表示するビューアです。Gallery、Deck、XYZ Plot、共通メディアカードから利用できます。

キーボード操作、スワイプ、前後ボタンに対応し、Lightbox 内でもコンテキストメニューが使えます。

### Dialog

Drawer は `showAlert`、`showConfirm`、`showPrompt`、`showDialog` を共通ダイアログとして提供します。設定、検索インデックス、確認操作、外部ガジェットのフォームなどで使われます。

### Image Picker

メディアファイル選択用のモーダルピッカーです。モデルプレビューに output 画像を設定するような場面で使われます。

## Settings

Settings では Drawer 全体の設定やメンテナンス操作を扱います。

主な項目は次の通りです。

- テーマとアクセントカラー
- 辞書の有効/無効
- ユーザー辞書、ワイルドカード、Danbooru 辞書の管理
- 検索インデックスの作成、同期、自動同期
- キャッシュ削除

## サードパーティ拡張

ComfyUI-Drawer は単なるビルトイン UI 集ではなく、外部カスタムノードが乗れる小さなプラットフォームとして設計されています。

### JavaScript ガジェット

外部カスタムノードは、`custom_nodes/*/web/js/` に JavaScript を置くことで Drawer ガジェットを登録できます。`window.ComfyDrawer` から `GadgetBase`、`bus`、`bridge`、`settings`、`dict`、`showDialog`、`contextMenu` などの公開 API を利用します。

単純なガジェットは 1 ファイルで完結できます。詳しい API は `GADGET_API.md` を参照してください。

### Python メタデータ拡張

Gallery のメタデータ処理は、Python 側の登録 API で拡張できます。

- metadata provider: 埋め込みメタデータ以外の raw metadata を返す
- index contributor: raw metadata 内のカスタムメタデータを Drawer の検索インデックス用フィールドへ変換する
- metadata panel contributor: カスタムメタデータをメタデータ表示パネルへ表示内容を追加する
- dictionary provider: 検索やプロンプト用の補完候補を提供する

重要なのは、Drawer がサードパーティの保存形式を指定しないことです。独自メタデータを `workflow.extra` に保存していても、別ファイルやデータベースで管理していても構いません。Drawer は raw metadata を受け取り、contributor/provider が返す Drawer 標準形式だけを使います。

検索 contributor と辞書 provider は別物です。検索 contributor はインデックスに値を入れるためのもの、辞書 provider は補完候補を表示するためのものです。大抵の場合両方登録すると思いますが、片方だけを登録することも可能です。

## モバイル利用

Drawer はモバイルや小画面でも使えることを重視しています。下部 Drawer に操作を集めることで、ノードグラフ全体を細かく触らなくても、主要な操作を進められます。

モバイルで特に使いやすい機能は次の通りです。

- Deck でよく触るパラメータだけを操作
- Gallery で生成結果を確認、検索、削除、ワークフローを開く
- Model Viewer でモデルや LoRA を探す
- Lightbox で画像や動画を全画面確認
- Context Menu をロングタップで開く
- Settings で辞書や検索インデックスを管理

モバイルでは画面サイズやブラウザ UI の影響を受けやすいため、ポップアップ類は画面内に収まるように調整されています。

## よくある質問

### Drawer は APP mode の代替ですか？

完全な代替ではありません。APP mode はワークフローをアプリのように使う方向ですが、Drawer はノードグラフを開いたまま編集・検証・試行錯誤するための操作面です。

### 専用ノードを置かないと使えませんか？

多くの機能は専用ノードなしで使えます。Gallery、Model Viewer、Lightbox、Context Menu、検索などはワークフローにノードを追加しなくても動きます。

Deck を整理して使う場合は、ノードタイトルのマーカーや DrawerControls が便利です。XYZ Plot も専用 XYZ ノードは不要です。現在のワークフロー内のウィジェットを対象にします。排他スイッチを活用するとより幅が広まります。

### 検索インデックスは必須ですか？

ファイル名検索だけなら必須ではありません。prompt、workflow、ノード、カスタムメタデータを検索したい場合は検索インデックスが必要です。

### 検索インデックスは自動で作られますか？

いいえ。大量ファイル環境で勝手に重い処理を始めないよう、ユーザーが明示的に作成します。作成前には簡易計測と確認が入ります。

### サードパーティの独自メタデータは検索できますか？

できます。ただし、そのメタデータをどう検索インデックスへ変換するかを知っている index contributor が必要です。独自保存形式を Drawer 形式へ移行する必要はありません。

### 画像に埋め込まれた workflow は開けますか？

Gallery や Context Menu から、workflow metadata を持つ画像をワークフローとして開けます。metadata provider がある場合は、埋め込み以外の raw metadata も provider-first で扱えます。

### コメントやワイルドカードは出力画像の metadata に残りますか？

コメントは実行用プロンプトから除去されますが、workflow metadata には保持されます。ワイルドカード展開結果も metadata 側で確認できるように扱われます。
